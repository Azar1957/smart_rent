# Архитектура Smart Rent

## Обзор

Smart Rent — двухслойная система:

```
┌──────────────────────────────────────────────────────────────────────────┐
│ Браузер (responsive UI, RU/EN/ES)                                        │
└──────────────────────────────┬───────────────────────────────────────────┘
                               │ HTTPS / JSON
┌──────────────────────────────▼───────────────────────────────────────────┐
│ Frontend: Next.js 14 (App Router) + TS + Tailwind                        │
│ - Серверный рендер страниц-витрин (SSR)                                  │
│ - Клиентские страницы дашбордов с SWR                                     │
│ - Прокси /api/iris/* → IRIS REST (через rewrite)                         │
└──────────────────────────────┬───────────────────────────────────────────┘
                               │ HTTP (внутренняя docker-сеть)
┌──────────────────────────────▼───────────────────────────────────────────┐
│ Ядро: InterSystems IRIS Community Edition                                │
│ - REST API (%CSP.REST) — диспетчер SmartRent.REST.Dispatch               │
│ - Сервисный слой — SmartRent.Service.*                                   │
│ - Модель данных — SmartRent.Model.* (persistent classes)                 │
│ - Хранилище — globals в namespace SMARTRENT (durable volume)             │
└──────────────────────────────────────────────────────────────────────────┘
```

## Контейнеры

| Контейнер             | Образ                                | Порты                    | Назначение |
|-----------------------|--------------------------------------|--------------------------|------------|
| `smartrent-iris`      | собирается из `iris/Dockerfile`      | `1972` (SQL), `52773` (Web) | СУБД + бэкенд |
| `smartrent-frontend`  | собирается из `frontend/Dockerfile`  | `3000`                   | Веб-интерфейс |

Том `iris-data` примонтирован в `/durable` контейнера IRIS — при пересборке образа данные не теряются.

## Безопасность (текущее состояние)

- Пароли пользователей хранятся как `SHA-512(пароль + случайная соль)` (см. `SmartRent.Model.User`).
- Сессионные токены — 256-битные случайные значения (`SmartRent.Service.Auth:GenerateToken`),
  передаются клиентом в `Authorization: Bearer <token>`.
- TTL токена — 24 часа, поддержка `revoked` для logout.
- Двухфакторная аутентификация подготовлена в модели (`TwoFactorSecret`, `TwoFactorEnabled`),
  включается на этапе доработки.
- CORS включён глобально в `SmartRent.REST.Dispatch.OnHandleCorsRequest`. В проде сузить до конкретных origin.
